# スマホ改造ガイド

このガイドでは、**Sony Xperia XZ2 Premium (SOV38)** のブートローダーアンロックからroot化までを、初心者でもわかるようにステップバイステップで解説します。

## 対象端末

このガイドはAU版SOV38で実際に行った作業をベースにしています。

| 項目 | 値 |
|------|-----|
| 機種名 | Sony Xperia XZ2 Premium |
| 型番 | SOV38 (AU版) |
| コードネーム | aurora_kddi |
| SoC | Qualcomm Snapdragon 845 |
| Android | 10 |

:::tip 他の国内版でも動作する可能性あり
このガイドはAU版 (SOV38) で検証していますが、**docomo版やSoftBank版など他の国内キャリア版XZ2 Premium**でも同様の手順で動作すると考えられています。xperableのブートローダーアンロックは国内キャリア版Xperia全般に対応しています。
:::

## このガイドでできるようになること

- **ブートローダーアンロック (BLU)** — 改造の第一歩。カスタムROMやroot化の前提条件
- **root化 (Magisk)** — 管理者権限を取得し、システムを自由にカスタマイズ
- **カスタムリカバリ (TWRP)** — バックアップや高度な操作が可能に

## 改造の全体像

```
Step 1: 準備（ツール導入・バックアップ）
    ↓
Step 2: ブートローダーアンロック（BLU）
    ↓
Step 3: カスタムリカバリ（TWRP）導入 + root化（Magisk）
    ↓
完了！ 自由にカスタマイズ可能に
```

## 改造前に知っておくべきこと

### メリット

- システムレベルのカスタマイズが自由に
- 広告ブロック、テーマ変更、CPU制御など
- 不要なプリインアプリの削除
- カスタムROMで最新Androidを使える可能性

### デメリット・リスク

:::danger 改造のリスク
以下のリスクを理解した上で進めてください：

- **メーカー保証が無効** になる
- **文鎮化（起動不能）** のリスクがある
- **カメラのDRM (Suntory/CKB) が失われる** — Sony独自の画像処理が無効化、画質が低下する
- **Widevine L1 → L3** に低下 — Netflix等のHD/4K再生ができなくなる
- **おサイフケータイ / FeliCa** が使えなくなる場合がある
- 操作を誤ると **復旧不可能** になることもある
:::

## SOV38の起動モードとLEDランプの意味

SOV38には複数の起動モードがあり、**LEDの色**で現在の状態がわかります。改造中に何度も使うので覚えておいてください。

### LEDランプの色

| LEDの色 | 意味 |
|---------|------|
| **緑** 点灯 | S1/XFLモード（フラッシュモード）に入っている |
| **青** 点灯 | 正規fastbootモード に入っている |
| **白** 点灯 | 充電中（電源オフ時） |
| **緑** 点滅→消灯 | 通常起動中（Androidが起動するとLEDは消える） |
| **緑→赤** 繰り返し | ブートループ（起動に失敗して再起動を繰り返している） |
| **赤** 点灯 | 起動失敗で停止している |

### 起動モード一覧

SOV38には3つの起動モードがあります：

| モード | 入り方 | 用途 | LED |
|--------|--------|------|-----|
| **通常起動** | 電源ボタン | Androidを起動 | 緑→消灯 |
| **S1/XFLモード** | 電源オフ → **Vol DOWN + USB接続** | ファームウェアフラッシュ、xperableでのBLU | 緑 |
| **正規fastboot** | `adb reboot bootloader` | TWRP起動、boot image書き込み | 青 |

:::warning S1モードと正規fastbootは別物！
- **S1モード** (緑LED): ファームウェアの書き込み専用。`fastboot boot` は**使えない**（"Command not supported"になる）
- **正規fastboot** (青LED): `fastboot boot` や `fastboot flash` が使える。**Androidが起動している状態からのみ**入れる

文鎮化してAndroidが起動しない場合、正規fastbootには入れません。S1モードで復旧作業を行う必要があります。
:::

### 各モードへの入り方

#### S1モード（緑LED）
1. 電源を完全にオフにする
2. **ボリュームDOWNを押しながら**USBケーブルをPCに接続
3. 緑LEDが点灯すれば成功

#### 正規fastboot（青LED）
```bash
# Androidが起動している状態で
adb reboot bootloader
# 青LEDが点灯すれば成功
```

#### 強制シャットダウン
起動しない・フリーズした場合：
- **電源ボタンを10秒以上長押し**（振動するまで）

## 用語集

| 用語 | 意味 |
|------|------|
| **ブートローダー (BL)** | スマホの電源を入れたとき最初に動くプログラム。OSを起動する役割 |
| **BLU (BL Unlock)** | ブートローダーのロックを解除すること。改造の第一歩 |
| **root化** | スマホの管理者権限（root）を取得すること。PCでいう「Administrator」 |
| **Magisk** | systemを書き換えずにrootを取得するツール。SafetyNetも回避可能 |
| **TWRP** | カスタムリカバリ。バックアップやZIPの書き込みができる |
| **文鎮化（ブリック）** | スマホが起動しなくなること。「高級な文鎮」になる |
| **TAパーティション** | Sony端末固有の領域。DRMキーやシリアル番号などデバイス固有情報を保存 |
| **S1/XFLモード** | Sonyのフラッシュモード。ファームウェア書き込みやTA操作に使う |
| **adb** | PCからスマホを操作するツール (Android Debug Bridge) |
| **fastboot** | ブートローダーモードでPCからスマホを操作するツール |
| **Suntory** | Sony独自のDRMエンジン。BLアンロックで消失する |
| **CKB** | Sony独自の画像処理エンジン。Suntoryに依存しておりBLU後は動作しない |

## 必要な環境

- **PC**: Windows / macOS / Linux（このガイドではmacOSを使用）
- **USBケーブル**: USB-C（SOV38側）で接続できるもの
- **Python 3.8以上**: xperableやsxflasherの実行に必要
- **安定したインターネット接続**
- **時間**: 初回は2〜3時間見ておくと安心
