# SOV38 Helper Toolkit

「コマンドとか全然わからない」という人でも、画面の指示に従うだけでBLアンロック〜root化まで進められるツールです。

xperable（ブートローダーアンロックの本体）を直接使うにはコマンドの知識が必要ですが、このヘルパーを使えば **番号を選ぶだけ** で自動的に必要な処理を実行してくれます。

## まず何をすればいいの？

### 1. ツールをダウンロード

PCでターミナル（コマンドを打つ画面）を開いて、以下をコピペします。

```bash
git clone https://github.com/hirorogo/xperable
cd xperable
```

:::info ターミナルの開き方
- **Mac**: Spotlight（⌘+Space）で「ターミナル」と検索
- **Windows**: スタートメニューで「PowerShell」と検索
- **Linux**: Ctrl+Alt+T
:::

### 2. xperable本体をビルド

```bash
make
```

これでxperableの実行ファイルが作られます。エラーが出る場合はgccなどの開発ツールが必要です。

:::tip Macの場合
初回だけ `xcode-select --install` が必要かもしれません（「コマンドライン開発ツール」のインストール）。
:::

### 3. ヘルパーを起動

```bash
python3 sov38_helper.py
```

こんなメニューが出ます：

```
  ╔══════════════════════════════════════════╗
  ║   SOV38 Helper Toolkit v1.1.0           ║
  ║   Sony Xperia XZ2 Premium (SOV38)       ║
  ╚══════════════════════════════════════════╝

  1) 環境チェック
  2) デバイスステータス確認
  3) TAパーティション バックアップ
  4) boot image バックアップ
  5) ブートローダーアンロック (BLU)
  6) Magisk root化
  7) フルガイド (1→5→6 を順番に実行)
  8) エクスプロイト リトライテスト
  q) 終了

  → 選択:
```

番号を入力してEnterを押すだけです。

## 各メニューの説明

### 1) 環境チェック

「adb」「fastboot」「python3」「xperable」が入っているか自動で確認します。足りないものがあれば何を入れればいいか教えてくれます。

**初めての人はまずこれを実行してください。**

### 2) デバイスステータス確認

SOV38をUSBでつないだ状態で実行すると、今の端末の状態を表示します：
- 機種名・Androidバージョン
- ブートローダーがロックされているかどうか
- root権限があるかどうか
- Magiskのバージョン

### 3) TAパーティション バックアップ

:::danger 一番大事な機能
TAパーティションには端末固有のデータが入っています。これを失うと**二度と元に戻せません**。BLアンロックの前に必ず取ってください。
:::

root権限がある場合はadb経由で自動ダンプします。root権限がない場合（初回BLU前）は、xperableが自動でバックアップしてくれるのでそちらに任せることもできます。

### 4) boot image バックアップ

現在のboot パーティションを保存します。Magiskでパッチする前のオリジナルを残しておくと、何か問題が起きた時に戻せます。

### 5) ブートローダーアンロック (BLU)

一番メインの機能。画面の指示に従うだけでBLアンロックが完了します。

やることの流れ：
1. リスクの説明を表示 → 同意するか確認
2. 「S1モードにしてください」と案内（やり方も表示される）
3. xperableを自動実行（失敗したら自動リトライ）
4. 再起動の案内
5. アンロックされたか自動確認

**自分でxperableのコマンドを打つ必要はありません。**

### 6) Magisk root化

BLアンロック後に使います。boot imageの取得 → Magiskでパッチ → 書き込みまで案内してくれます。

### 7) フルガイド

環境チェック → BLアンロック → root化 を一気に通しで実行します。初めての人はこれがおすすめ。

### 8) エクスプロイト リトライテスト

上級者向け。xperableの各オプション（`-0`, `-2`, `-4`, `-5`）を自動リトライで試せます。

## コマンドライン引数（メニューを使わない方法）

メニューを開かずに直接実行もできます：

```bash
python3 sov38_helper.py --check     # 環境チェックだけ
python3 sov38_helper.py --status    # デバイス確認だけ
python3 sov38_helper.py --backup    # バックアップだけ
python3 sov38_helper.py --unlock    # BLアンロックだけ
python3 sov38_helper.py --magisk    # root化だけ
```

## 自動リトライってなに？

xperableのエクスプロイトは **USBの通信タイミング** を利用しているので、1回で成功するとは限りません。むしろ何回か失敗するのが普通です。

このヘルパーは失敗したら自動で繰り返し試してくれます。

### 動き方

```
① まずデフォルト設定で20回試す
   [████████░░░░░░░░░░░░] #8 試行 8/20
   → 5回連続失敗するとUSBケーブル抜き差しを案内

② 20回やって全部ダメ → バッファサイズを変えてもう20回
③ それでもダメ → また別のサイズで20回
④ 全10サイズ × 20回 = 最大200回まで自動で試す
```

### バッファサイズって？

USBで大きなデータを送る時、OSが自動的にデータを小分けにして送ります。この「小分けの大きさ」がバッファサイズです。

```
例: 大きなデータを送る場合

  PC → [====16KB====] → SOV38   ← 16KBずつ送る
  PC → [====16KB====] → SOV38
  PC → [====16KB====] → SOV38
  PC → [====16KB====] → SOV38
```

この大きさによってエクスプロイトの成功率が変わります。PCのUSBドライバやケーブルによって最適な値が違うので、ヘルパーが自動で色々な値を試してくれます。

### 成功率を上げるコツ

- **短いUSBケーブルを使う** — 長いと電気的なノイズが増えて不安定に
- **USBハブを使わず直接つなぐ** — ハブ経由だと通信が不安定になりやすい
- **USB 2.0ポートを使う** — USB 3.0ポート（青いやつ）より2.0の方が安定する場合がある
- **PC再起動** — USBドライバの調子が悪い時はこれで直ることがある
- **別のPC** — USBコントローラの相性問題もある

## ログファイル

すべてのリトライ結果はJSONファイルに自動保存されます：

```
sov38_backups/
  exploit_logs/
    exploit_log_20260302_143022.json   ← 日時入りのログ
  TA_backup_20260302_120000.img       ← TAバックアップ
  boot_a_backup_20260302_120000.img   ← bootバックアップ
```

失敗が続いた時は、このログを見ればどんなエラーが多かったかわかります。

## うまくいかない時は

| 症状 | 対処 |
|------|------|
| `python3: command not found` | Python3がインストールされていない → [python.org](https://www.python.org/) からインストール |
| `make: command not found` | 開発ツール未インストール → Macなら `xcode-select --install`、Linuxなら `sudo apt install build-essential` |
| `xperable バイナリが見つかりません` | `make` を実行してビルドしてからもう一度 |
| `ADBデバイスが見つかりません` | USBデバッグがONか確認 / USBケーブルを変える / ドライバインストール（Windows） |
| `unauthorized` | スマホ画面に出る「USBデバッグを許可しますか？」をOKする |
| 200回リトライしても全部失敗 | ケーブルを変える / PCを変える / [Global版ファームを先に焼く](/docs/phone-mod/brick-recovery#補足-global化の正しい方法taを変更しない) |

:::tip Global版ファームを先に焼くと楽
国内版ブートローダーだとxperableの成功率が低い場合があります。[Global版ファームウェアを先にフラッシュ](/docs/phone-mod/brick-recovery#補足-global化の正しい方法taを変更しない)すれば、ブートローダーもGlobal版に置き換わり、デフォルト設定で通りやすくなります。
:::
