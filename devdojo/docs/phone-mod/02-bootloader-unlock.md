# ブートローダーアンロック

## 背景

Sony Xperiaのキャリア版（AU/docomo/SoftBank）はブートローダーがロックされており、カスタムROMやroot化ができない。公式アンロックサービスも日本キャリア版は対象外。

## xperable とは

[xperable](https://github.com/xperiofficial/xperable) は Sony Xperia のブートローダーを非公式にアンロックするツール。S1/XFL フラッシュモードでデバイスに接続し、TAパーティションの `rooting_status` を書き換える。

### 動作原理

1. デバイスをS1モード（フラッシュモード）で接続
2. TAパーティションから現在の `rooting_status` を読み取り
3. `rooting_status` を `2`（アンロック済み）に書き換え
4. **自動的にTA.imgのバックアップを作成**（これが後で命を救う）

### 実行

```bash
# S1モードで接続（電源オフ → ボリュームDOWN押しながらUSB接続 → 緑LED）
python xperable.py
```

## アンロック後に失われるもの

:::danger 重要
ブートローダーアンロックにより、以下が**不可逆的に**失われる：

- **Suntory DRMキー** — カメラのCKBエンジンが動作しなくなる
- **Widevine L1** — Netflix等のHD再生が不可に（L3にダウングレード）
- **FIDO認証キー**
- 起動時に「ブートローダーがアンロックされています」警告が表示
:::

### rooting_status の値

| 値 | 意味 |
|----|------|
| 0 | ロック済み（出荷時） |
| 2 | アンロック済み |
| 5 | アンロック済み（別のフラグ） |

`rooting_status` はTAパーティションに保存され、システムプロパティ（`ro.boot.*`）とは**独立**している。`resetprop` でプロパティを書き換えても、TAの値は変わらない。

## CDA_NR の変更（文鎮化の原因）

ブートローダーアンロック後、xperableでCDA_NR（カスタマイゼーションコード）を変更しようとした。

```
変更前: 1314-0765 (KDDI JP)
変更後: 1311-7853 (Global)
```

### なぜ失敗したか

CDA_NRはTAパーティション内の **20箇所以上** に存在する。xperableは `0x020c4d` の1箇所のみを変更したため、整合性が崩れた。

```
TAパーティション内のCDA_NR出現箇所: 20箇所以上
xperableが変更した箇所: 1箇所のみ
→ 整合性崩壊 → 起動不能（文鎮化）
```

### 症状

- S1/XFLフラッシュモードでしか起動しない
- Android、fastboot モードに到達できない
- LEDが赤点灯のまま
